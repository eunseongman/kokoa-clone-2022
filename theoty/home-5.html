transition - 상태의 "변화"를 애니메이션으로 만드는 방법
- transition 이라는 속성은 state가 없는 요소에 붙어야 함 (root, elment)
- state에 변화의 결과가 명시되어 있어야 함
- transition을 사용하지 않는 차이점은 그저 순식간에 변하기만 함
(ex - transition: (대상) (시간) (ease-in-function) )
- ease-in-function : 애니메이션이 어떻게 변할지 명시
(시작 ~ 종료의 속도 구분 : linear, ease-in, ease-out, ease-in-out)
+ cubic-bezier() : 사용자 설정 속도

transformation - 한 요소를 "변형" 시키는 방법
- ex : rotateX,Y,Z / translateX,Y,Z / scale / skew ... 등등 존재
- 주변 box element(sibling)를 변형시키지 않고 변형시킴
(margin, padding이 적용되지 않음)
- transition과 같이 연계하여 사용하기 좋음

animation - state 변화가 없더라도 애니메이션으로 만드는 방법
- 정의 : @keyframes name { from {start} to {end}}
- 적용 : animation: name time ease-in-function (infinite);
(끝에서 다시 시작으로 점프하는 것을 해결할 순 없을까? )
- 변형 : @keyframes name { 0% {} 50% {} 100%{} }
(퍼센티지를 추가하여 단계를 더 설정할 수 있음)

media query - css를 이용하여 스크린의 사이즈를 알 수 있는 방법
- 적용 : @media screen and (조건) {내용}
- and () 를 추가로 덧 붙여 조건을 여러개 줄 수 있음
- 조건 : min/max-width , orientation: landscape/portrait ...
(+ min/max-device-width : only phone)
- screen 대신 print : 해당 페이지를 인쇄할 때 보여주는 화면

git - 파일의 변경내용을 계속 추적(tracking)하는 것 (version control system)
github - 파일 내역과 파일들을 올려주는 공간 (사업체)
- repositoy : 코드의 번경 내역과 그 히스토리를 갖고 있는 폴더
- 각 프로젝트들은 각각의 commit (저장)을 확인할 수 있고, 어떤 것을 수정했는지 보여줌
